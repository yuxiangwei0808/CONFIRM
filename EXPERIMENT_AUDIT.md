# Experiment Audit Report

**Date**: 2026-06-16  
**Auditor**: GPT-5.5 xhigh cross-model reviewer  
**Project**: CONFIRM / NeuroScientistAgents  

## Overall Verdict: WARN

## Integrity Status: warn

The audit found no evidence of fake ground truth derived from model outputs, no self-normalized performance
scores, and no mismatched main benchmark numbers. The main risk is scientific scope: many labels are
proxy/stability/stress-test labels rather than official external ground truth, so broad paper claims must stay
qualified until the literature-label pass and benchmark expansion are complete.

## Checks

### A. Ground Truth Provenance: WARN
The benchmark-ready runner derives labels from `expected_label` / `claim_type` in the claim inventory. Injected
null labels are generated by deterministic runner logic, and positive/fragile labels are currently proxy labels
that need external literature or expert adjudication. This is explicitly recorded in the lockfile, so it is not
an integrity failure.

### B. Score Normalization: WARN
No max/min/mean normalization over model outputs was found. TPR/FCR use raw counts and denominators. The warning
is that `power.py` can fall back to observed standardized effects when no external reference effect is supplied;
those cases should be described as winner's-curse risk or supplied with predeclared MDE/reference effects.

### C. Result File Existence / Claim Match: WARN
Main benchmark-ready numbers matched the result files at audit time. A later cleanup replaced stale tracker
paths with the current staged index:
- current result index: `docs/STAGED_RESULTS_INDEX.md`
- current primary result: `review-stage/combined-label-aware-combat/combined_benchmark_results.json`
- no-shared-feature claims are demoted by the benchmark-ready builder

### D. Dead Code Detection: WARN
The audit found duplicated brain-wide multiverse code and an unused `_raw_effect_map` helper. Post-audit fixes:
- `confirm.multiverse.run_brainwide_multiverse` is now the canonical shared implementation
- `run_benchmark_ready.py` calls the shared implementation
- the generic agent path calls the shared implementation
- `_raw_effect_map` was removed
- stale generated `src/confirm.egg-info/` metadata was removed

The injected-null builders remain intentionally: they are not used by the current prepared-data runner, but they
are source utilities for future masked-trap generation.

### E. Scope Assessment: WARN
The current hardened wave is 20 runnable fMRI claims over 8 fMRI cohorts, with 4 positive/stability labels and
16 null/fragile/proxy-negative labels, under no-harmonization and ComBat settings. This supports an internal
benchmark-ready pilot, not broad cross-disease robustness.

### F. Evaluation Type: proxy / synthetic / simulation mix
- fMRI positive and fragile labels: real data with proxy/stability labels
- injected nulls: synthetic proxy labels over real imaging features
- hardening FDR trap: synthetic proxy over HCP FC features
- power/MDE trap and unit tests: simulation-only
- human evaluation: not yet included

## Claim Impact

- Supported narrowly: the benchmark-ready framework runs end-to-end, emits audit/lockfile receipts, evaluates
  no-harmonization and ComBat settings, and catches injected FDR/MDE traps.
- Needs qualifier: FCR/TPR are against proxy/stress-test labels, not official disease-biomarker ground truth.
- Unsupported for now: broad claims that CONFIRM robustly reduces false neuroimaging discoveries across disease
  domains.

## Post-Audit Label Update

The first-pass literature ledger recommends moving RSFC-cognition claims out of the false-confirmation bucket
and into `small_positive_expected` / `small_positive_candidate`. Marek et al. 2022 supports small but
replicable RSFC-cognition associations at large N, so confirmed cognition claims should be reported separately
until the final label table is adjudicated.
